# AWS Lambdaとは

## 学習目的
- AWS Lambdaの基本的な考え方を理解する
- サーバーレスとは何かを自分の言葉で説明できるようになる
- TerraformでLambdaを扱う前提知識を整理する

## AWS Lambdaの概要
AWS Lambdaは、サーバーの構築や管理を意識することなくコードを実行できる
AWSのサーバーレスコンピューティングサービスである。

実行の単位は「関数」で、イベントをきっかけに処理が実行される。
処理が実行された時間分だけ課金されるため、常時起動のサーバーとは異なる
コストモデルになっている。

## サーバーレスの考え方
サーバーレスとは「サーバーが存在しない」という意味ではなく、
インフラの管理（OS、パッチ適用、スケーリングなど）を
クラウド事業者に任せる考え方である。

利用者は以下の点に集中できる。
- ビジネスロジックの実装
- イベント設計
- 権限やセキュリティの設計

## Lambdaでできること
- APIのバックエンド処理
- 定期実行のバッチ処理
- S3やSQSなど他AWSサービスと連携したイベント処理
- 軽量なデータ変換や通知処理

## Lambdaの制約・注意点
- 実行時間に上限がある（長時間処理には不向き）
- ローカルファイルは永続化されない
- 実行環境は毎回同じとは限らない

これらの制約を理解したうえで設計する必要がある。

## 学習メモ
- 「サーバーレス＝楽」ではなく、設計の考え方が重要
- イベント駆動で考える必要がある
- Terraformと組み合わせることで再現性の高い構成管理ができそう
